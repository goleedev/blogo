# 💁🏻 이진 탐색 트리란

![BST](/static/images/bst2.png)

**`이진 탐색 (Binary Search)`과 `연결 리스트 (Linked List)`를 결합한 트리 형태 자료구조**입니다. 이진 탐색의 **빠른 탐색 속도**와 연결 리스트의 **빠른 추가 및 삭제라**는 장점을 결합하였습니다. 이진 탐색 트리에서는 **왼쪽 자식노드는 부모보다 크기가 작고, 오른쪽 자식노드는 부모보다 크기가 큽니다**. 이때 노드는 **중복되면 안되며, 자식노드는 부모노드와 같이 이진 탐색 트리 구조**여야 합니다.

---

# 🛠️ 이진 탐색 트리 사용 방법

## 추가 (Insert)

- 추가하고자 하는 숫자가 부모노드 숫자보다 작으면, 왼쪽에 새로운 노드를 삽입합니다.
- 추가하고자 하는 숫자가 부모노드 숫자보다 크면, 오른쪽에 새로운 노드를 삽입합니다.

## 검색 (Contains)

- 찾으려는 값이 부모노드에 있으면 true를 리턴합니다.
- 찾으려는 값이 부모노드에 없으면 크기를 판단합니다.
- 크기가 부모노드 값보다 작으면 왼쪽에서 탐색하고, 크면 오른쪽에서 탐색합니다.

## 중외순환 (Inorder)

> 왼쪽 자식노드 ➡️ 자신노드 ➡️ 오른쪽 자식노드 순으로 정렬하며, 숫자가 작은 것 부터 큰 순으로 나열됩니다.

- 왼쪽 자식노드에 대해 노드가 존재하는지 확인하고 있으면 출력합니다.
- 현재 노드가 존재하는지 확인하고 있으면 출력합니다.
- 오른쪽 자식노드에 대해 노드가 존재하는지 확인하고 있으면 출력합니다.
