# 💁🏻 Reflow란

![Repaint Reflow](/static/images/reflow-repaint2.png)

화면이 수정될 때 렌더링 과정을 최적화하는 방법은 두가지가 있는데요. 그 중에 하나가 바로 화면을 다시 그리는 과정인 `Reflow (Layout)`입니다. Reflow는 화면 구조 (Layout)가 변경되었을 때, 뷰 포트 내에서 **렌더 트리의 노드의 정확한 위치와 크기를 계산하는 과정**을 말합니다.

## Reflow가 발생하는 경우

> 아래의 경우에 해당되지 않더라도, 화면의 조가 변경되었다면 Reflow가 발생합니다.

- **DOM 노드의 추가 및 제거**
- **DOM 노드의 위치 변경**
- **DOM 노드의 크기 변경** (margin, padding, border, width, height 등)
- CSS3 **애니메이션과 트랜지션**
- **폰트 및 텍스트 내용 변경**
- **이미지 크기 변경**
- `offset`, `scrollTop`, `scrollLeft`과 같은 **계산된 스타일 정보 요청**
- **페이지 초기 렌더링**
- **윈도우 리사이징**

## Reflow의 최적화

- **인라인 스타일**을 사용하지 않습니다.
- 애니메이션이 있는 노드는 **position을 `fixed` 또는 `absolute`로 지정**합니다.
- **퀄리티 및 퍼포먼스의 타협점**을 찾습니다.
- **`<table>` 레이아웃**을 피합니다.
- **IE의 CSS 표현식**을 사용하지 않습니다.
- CSS **하위 선택자를 최소화**합니다.
- **숨겨진 노드의 스타일을 변경**합니다.
- 클래스를 혹은 cssText 사용하여 **한 번에 스타일을 변경**합니다.
- **DOM 사용을 최소화**합니다.
- **캐시**를 활용합니다.

# 🎨 Repaint란

화면이 수정될 때 렌더링 과정을 최적화하는 나머지 방법이 바로 `Repaint (Redraw)`입니다. Repaint는 **화면에 변화가 있을 때 화면을 그리는 과정**인데요. 화면의 구조가 변경되었을 때에는 **Reflow 과정을 거쳐 화면 구조를 다시 계산한 후, Repaint 과정을 통해 화면을 다시 그리는 것**입니다. 즉, 화면의 구조가 변경되었을 때에는 **Reflow와 Repaint 모두 발생**합니다. 따라서 **화면의 구조가 변경되지 않는 화면 변화의 경우에는 Repaint만 발생**합니다.
