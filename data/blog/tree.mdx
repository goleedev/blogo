# 🌴 트리란

![트리](/static/images/tree.png)

`트리 (Tree)`는 **계층 구조 (Hierarchical Structure)로 자료를 저장하는 자료구조**입니다. 여기서의 계층 구조란 트리를 구성하는 **노드가 부모-자식 (Parent-Child) 관계**라는 것을 뜻하는데요. 즉, 특정 부모 노드 하나에 여러 개의 자식 노드들이 연결되는 구조를 뜻하는 것입니다. 또한, 구조로 노드들이 **연결된 모습이 마치 마치 나무**와 같아 트리라는 이름이 붙여졌습니다. 이러한 트리는 컴퓨터의 폴더 구조나 가족의 가계도와 같은 **계층적인 관계를 가진 자료**를 표현하고 싶은 경우에 사용되는 자료구조입니다.

# 📖 트리 관련 용어

![트리](/static/images/tree2.png)

**노드 (node)**: 여기서 말하는 노드란 트리의 구성 요소들을 말하며, 위의 그림에서 각 알파벳을 뜻합니다.

**간선 (edge)**: 노드와 노드를 연결하는 선을 말합니다.

**루트 노드 (root node)**: 위 그림에서의 최상위 노드, 즉 ‘A‘를 뜻합니다.

**단말 노드**: 아래로 연결되어 있는 노드가 없는 노드를 뜻합니다.

**내부 노드**: 단말 노드를 제외한 모든 노드를 뜻합니다. 이때, 루트 노드도 내부 노드라고 할 수 있습니다.

**부모 노드**: 자식의 부모, 즉 상위가 되는 노드를 뜻합니다.

**자식 노드**: 부모 노드의 하위 노드를 자식 노드를 말합니다.

**형제 노드**: 부모 노드가 같은 노드들을 모두 형제 노드라고 말하며, 서로 서로가 형제 노드가 됩니다.

**레벨**: 트리에서 각 층별로 숫자를 매겨서 이를 트리의 레벨이라고 표현합니다.

**높이**: 트리의 최고 레벨을 가리켜 높이라고 하며, 위 그림에서는 레벨이 0-3 이므로 높이가 3이 됩니다.
