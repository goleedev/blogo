# 💁🏻 HTTP란

[HTTP](https://www.golee.tech/blog/http)에 대해서는 이전 글에서 자세히 다루었으니 참고 해주시면 됩니다. 다시 한번 간단히 정리하자면, `HTTP`란 **Hypertext Transfer Protocol**의 약자로 서로 다른 시스템들 사이에서 **통신을 주고받게 해주는 가장 기초적인 프로토콜**을 말합니다.

`HTTP`는 웹 서핑을 하다보면 흔히 보게 되는 데, 이 HTTP에는 **문제점**이 있습니다.

---

# 🤔 HTTP의 문제점

`HTTP` 통신을 하게 되면 서버에서부터 브라우저로 전송되는 정보가 암호화 되어 있지 않으며, 반대의 경우에도 같은 문제를 가지고 있습니다. 즉, **데이터 전송 과정에서 데이터를 도난** 당할 수도 있다는 것인데요. 특히 **비밀번호와 같은 민감한 정보**가 암호화 되지 않은 채 돌아다닐 수 있는 것이기 때문에 큰 문제점이라고 볼 수 있습니다.

---

# 🔐 HTTPS란

`HTTPS`는 **Hypertext Transfer Protocol Secure**의 약자로 HTTP와는 다르게 `S` 하나가 더 붙어있는데요. 여기서 `S'` 바로 **Secure Socket Layer (SSL)** 의 S를 뜻합니다. 즉, 위에서 언급된 HTTP 문제점인 **보안이 강화된 버전**이라고 이해하시면 됩니다.

`HTTPS` 프로토콜은 이러한 SSL을 사용함으로써 이러한 보안 문제를 해결한 것인데요. **SSL은 서버와 브라우저 사이의 암호화**된 연결을 도와줄 뿐만 아니라 **정보 도난을 방지**주고,, **`HTTP request, response`를 암호화 처리** 해줍니다. 이처럼 사용자가 **사이트에 제공하는 정보를 암호화**해 주는데, 데이터를 전송하는 중간에 도난 당하더라도 이미 **데이터가 암호화 되어 있기 때문에 해독이 불가능**하여 안전합니다.

또한, `HTTPS`는 그 외에도 **TLS (전송 계층 보안) 프로토콜**을 통해서도 보안을 유지하는데요. TLS는 SSL을 계승하여 만들어졌다고 하며, 표준화 기구인 IETF의 관리로 변경되면서 명칭이 변경된 것이라 합니다. 엄밀히 따지면 두 가지의 차이점이 존재하기는 하지만, 최근에는 SSL이라고 통칭하고 있다고 합니다.

---

# 🏄🏻 HTTPS 보안 외의 장점

보안 뿐만이 아닌 **검색 엔진 최적화 (SEO)** 에 관련해서도 큰 장점을 가지고 있는데요. 아는 동일한 키워드의 사이트가 존재한다고 했을 때, 키워드 검색 시 **상위 노출되는 기준 중 하나가 보안 부분**이기 때문입니다. 이는 결국 **사용자들이 더 안전하다고 생각하는 사이트를 조금 더 자주 방문하는 경향**이 반영된 것인데요. 이처럼 추가적인 장점이 탑재된 `HTTPS`가 당연하게 HTTP에 비해 속도 면에서는 다소 느리다고 볼 수 있지만, 최근에는 인터넷과 브라우저의 발달로 큰 차이를 느낄 수 없다고 합니다.
