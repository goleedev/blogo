# 💁🏻 CSS란

CSS는 `Cascading Style Sheets`의 약자로, HTML의 각 요소들의 style을 정의하여 **화면 등에 어떻게 렌더링하면 되는지 브라우저에게 설명하기 위한 언어**입니다.

초기에는 웹의 외형을 구현하면서 HTML이 가진 정보 구조를 해치기까지 했지만, HTML5이 널리 사용되면서 **HTML은 정보와 구조화, CSS는 styling의 정의**라는 명확한 구분이 이루어지고

HTML이 가진 정보 구조를 해치지 않으면서 웹 사이트를 디자인할 수 있게 되었습니다.

---

# 👷🏻 CSS의 기초

CSS는 크게 `선택자 (selector)`와 `선언부 (declaration block)`로 이루어지며, 그리고 선언부는 다시 `속성 (property)`과 `속성 값 (value)`으로 이루어집니다.

선언부는 **세미콜론 (;)으로 속성과 속성 값을 구분하여 여러 개의 선언(declaration)을 지정**할 수 있습니다.

간단히 아래와 같이 정리할 수 있습니다.

```

- 선택자
- 선언부
    - 속성: 속성 값; // 선언 1
    - 속성: 속성 값; // 선언 2
        .
        .
        .

```

위에서 설명한 것처럼 **선택자에 스타일링에 대한 선언을 부여**하면 됩니다.

```css
p {
  color: black;
}

.class {
  color: red;
}

#id {
  color: yellow;
}
```

# 📐 CSS 작성 팁

## Boolean 속성은 값을 따로 명시하지 않음

`selected`, `disabled`, `checked` 등의 Boolean 속성은 값을 따로 명시하지 않습니다.

```

<!-- Bad -->
<input type="text" disabled=true>

<!-- Good -->
<input type="text" disabled>

```

## 엔티티 참조는 사용하지 않음

`&rdquo;`, `&#x263a;` 등의 엔티티 참조를 사용할 필요가 없습니다.

```

<!-- Bad -->
The currency symbol for the Euro is `“&eur;”`.

<!-- Good -->
The currency symbol for the Euro is “€”.

```

## CSS/자바스크립트 파일을 불러오는 경우 `type`을 명시하지 않음

type 속성의 기본값이 각각 `text/css`, `text/javascript`로 지정되어 있으므로, 따로 속성값을 명시하지 않습니디.

```

<!-- Bad -->
<link rel="stylesheet" href="//uicdn.toast.com/tui.chart/latest/tui-chart.min.css" type="text/css">

<!-- Good -->
<link rel="stylesheet" href="//uicdn.toast.com/tui.chart/latest/tui-chart.min.css">

<!-- Bad -->
<script src="//uicdn.toast.com/tui.chart/latest/tui-chart.min.js" type="text/javascript"></script>

<!-- Good -->
<script src="//uicdn.toast.com/tui.chart/latest/tui-chart.min.js"></script>

```

## border가 없을 경우 none대신 0을 사용

```

/* Bad */
.no-border {border: none;}

/* Good */
.no-border {border: 0;}

```
