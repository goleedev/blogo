# 💁🏻 JSON이란

JSON은 `JavaScript Object Notation`의 약자로 **데이터를 저장하거나 전송할 때 많이 사용되는 경량의 데이터 교환 형식**입니다.

또한, JSON은 Javascript에서 **객체를 만들 때 사용하는 표현식**을 의미하며, 사람과 기계 모두 이해하기 쉽고 용량이 작아 최근에는 JSON이 XML을 대체해 **데이터 전송 등에 많이 사용**합니다.

간단히 말해, JSON은 어떠한 통신 방법도 프로그래밍 문법도 아닌 **단순히 데이터를 표시하는 데이터 포맷**입니다.

---

# ⚖️ JavaScript VS JSON

JSON은 **객체, 배열, 숫자, 문자열, 불리언과 null을 직렬화하기 위한 구문**으로, JavaScript 구문에 기반을 두고 있지만 분명한 차이점을 가지고 있습니다.

자바스크립트 Object는 **자바스크립트 Engine 메모리 안에 있는 데이터 구조**이며, JSON은 **객체의 내용을 기술하기 위한 text 파일**입니다.

이와 같은 이유로 JSON을 자바스크립트 Object로 파싱하기 위해서는 `JSON.parse()` 메서드가 필요하며, 자바스크립트 Object를 JSON으로 변환해주기 위해서는 `JSON.stringify()` 메서드가 필요합니다.

# ✨ JSON 특징

- **서버와 클라이언트 간의 교류**에서 일반적으로 많이 사용됩니다.

- 자바스크립트 **객체 표기법과 아주 유사**합니다.

- 자바스크립트를 이용하여 **JSON 형식의 문서를 쉽게 자바스크립트 객체로 변환**할 수 있습니다 (⚖️ JavaScript VS JSON 참고)

- JSON 문서 형식은 자바스크립트 **객체의 형식을 기반**으로 만들어졌습니다.

- 자바스크립트의 문법과 굉장히 유사하지만 **텍스트 형식**일 뿐입니다.

- 특정 언어에 종속되지 않으며, **대부분의 프로그래밍 언어에서 JSON 포맷의 데이터를 핸들링 할 수 있는 라이브러리를 제공**합니다.

# 🤷🏻 XML vs JSON

데이터를 나타낼 수 있는 방식은 여러가지가 있지만 대표적인 형식은 `XML`과 `JSON`입니다.

## XML

XML은 HTML을 기반으로 했기에 **데이터 값 양쪽으로 태그가 존재**합니다.

## JSON

JSON은 **중괄호({})로 표현하는 형식**으로 가지고 있고, 값을 ','로 구분짓습니다.
