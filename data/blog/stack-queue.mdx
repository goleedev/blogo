# 💁🏻 스택이란

![스택](/static/images/stack.webp)

스택은 `Last In First Out` 원칙으로 만들어진 자료 구조입니다. 즉, 가장 나중에 들어온 데이터가 먼저 나가야 그 전에 들어온 데이터들이 나갈 수 있는 구조로, 위의 그림에서처럼 **출입구가 동일한 것**이 특징입니다. 함수 실행 콘텍스트들이 쌓이는 `Call Stack`과 브라우저의 방문 기록이 쌓이는 `History Stack`이 대표적입니다.

## 스택의 속성

**top**: 스택 맨 위의 아이템을 뜻합니다.

## 스택의 메서드

- **push**: 스택의 맨 위에 아이템을 삽입합니다.
- **pop**: 스택 맨 위의 아이템을 제거하고 및 그 아이템을 반환합니다.
- **contains**: 해당 아이템이 스택에 존재하는지 확인합니다.
- **size**: 현재 스택에 있는 아이템의 총 개수를 반환합니다.

```javascript
class Stack {
  constructor() {
    this._arr = []
  }
  push(item) {
    this._arr.push(item)
  }
  pop() {
    return this._arr.pop()
  }
  peek() {
    return this._arr[this._arr.length - 1]
  }
}

const stack = new Stack()
stack.push(1)
stack.push(2)
stack.push(3)
stack.pop() // 3
```

# 🤲 큐란

![큐](/static/images/queue.webp)

큐는 `First In First Out` 원칙으로 만들어진 자료구조입니다. 즉, 먼저 들어온 데이터가 먼저 나가게 됩니다. 위의 그리모가 같이 큐는 **입구와 출구가 구분**되어 있기 때문에, 입구로 들어온 데이터는 반드시 반대쪽 출구로 나가야만 합니다. 자바스크립트 엔진에서 비동기 함수 실행 시 콜백들이 대기열로 들어오는 `Task Queue`가 대표적입니다.

## 큐의 속성

`first`: 큐 맨 앞의 아이템입니다.

## 큐의 메서드

- **enqueue**: 큐에 아이템을 추가합니다.
- **dequeue**: 큐 맨 앞의 아이템을 제거하고 및 그 아이템을 반환합니다.
- **contains**: 해당 아이템이 큐에 존재하는지 확인합니다.
- **size**: 현재 큐에 있는 아이템의 총 개수를 반환합니다.

```javascript
class Queue {
  constructor() {
    this._arr = []
  }
  enqueue(item) {
    this._arr.push(item)
  }
  dequeue() {
    return this._arr.shift()
  }
}

const queue = new Queue()
queue.enqueue(1)
queue.enqueue(2)
queue.enqueue(3)
queue.dequeue() // 1
```
