# 💁🏻 자바스크립트 자료형

JavaScript는 `원시 타입`과 `참조 타입`이라는 **두 가지 자료형을 제공**하며, Object를 제외한 모든 것들은 Primitive한 성격을 가지고 있습니다.

- **Primitive Type**: 데이터의 실제 값 할당
- **Reference Type**: 데이터의 위치 값만 할당

---

# 🗿 Primitive Type (원시 타입)

원시 타입의 데이터는 **변수에 할당이 될 때 메모리 상에 고정된 크기로 저장이 되고 해당 변수가 원시 데이터 값을 보관**합니다.

원시 타입 자료형은 모두 **변수 선언, 초기화, 할당 시 값이 저장된 메모리 영역에 직접적으로 접근**합니다.

즉, 변수에 새 값이 할당이 될 경우에는 할당된 메모리 블럭에 **저장된 값을 바로 변경**합니다.

## 원시 타입의 종류

- boolean
- number
- string
- null
- undefined

## 원시 타입의 변수 복사

각 변수 간에 원시 타입 데이터를 복사할 경우, **데이터의 값이 복사**됩니다.

```javascript
var x = 100
var y = x

x = 99

console.log(y) // 100;
```

위의 예시처럼 데이터의 값을 복사하기 때문에 console을 찍기 전,

x는 99로 바뀌었지만 **이전의 값인 100을 복사해두었기 때문에 최종적으로 100이 찍히는 것**을 볼 수 있습니다.

---

# 🏠 Reference Type (참조 타입)

참조 타입의 **데이터는 크기가 정해져 있지 않고 변수에 할당이 될 때 값이 직접 해당 변수에 저장될 수 없으며 변수에는 데이터에 대한 참조만 저장**됩니다.

변수의 값이 저장된 **힙 메모리의 주소 값을 저장하고 저장된 메모리 블럭의 주소**를 가지고 있으며, 자바스크립트 엔진이 **변수가 가지고 있는 메모리 주소를 이용해서 변수의 값에 접근**합니다.

## 참조 타입의 종류

- Object ( array, function, object )

## 참조 타입의 변수 복사

각 변수 간에 참조 타입 데이터를 복사할 경우, **데이터의 참조가 복사**됩니다.

```javascript
var x = { count: 100 }
var y = x

x.count = 99

console.log(y) // 99
```

변수 x와 y는 **동일한 참조**를 담고 있으며, 동일한 객체를 가리키게 됩니다.

---

# ⚖️ 원시 타입 VS 참조 타입

| **원시 타입 (Primitive Type)**      | **참조 타입(Reference Type)**           |
| ----------------------------------- | --------------------------------------- |
| 값의 저장 방법 하나의 데이터만 가짐 | heap에 데이터'들'을 저장 후 주소를 가짐 |
| 변수에 할당 시 값 (value)           | 주소 (reference)                        |
