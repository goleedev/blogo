# 💁🏻 Git이란

[Git](https://www.golee.tech/blog/git-note)에 대해 자세히 정리한 내용은 해당 글에서 참고하실 수 있습니다. 간단히 다시 말하자면, `Git`은 소스 코드 변경사항을 효율적으로 관리하기 위한 **분산형 버전 관리 시스템**입니다.

---

# 🐙 GitHub이란

`GitHub`은 **Git을 기반으로 하는 데이터 저장소**입니다.

---

# 📊 Git의 상태

![Git Stat](/static/images/github2.png)

- `Committed`: 데이터가 **로컬 데이터베이스에 안전하게 저장되었다는 것**을 뜻합니다.

- `Modified`: 수정한 파일을 아직 **로컬 데이터베이스에 커밋하지 않은 것**을 뜻합니다.

- `Staged`: 현재 **수정한 파일을 곧 커밋할 것**이라고 표시한 상태를 뜻합니다.

---

# 📚 관련 용어 정리

## Commit

깃에 데이터가 저장되어 온 기록 (파일 추가, 변경 등)을 남기는 행위를 뜻하며, 변경한 시간 순서대로 저장하고 영문과 숫자로 이루어진 40자리 이름으로 각 커밋 구분합니다.

## Repository

저장소를 뜻합니다.

## Remote Repository (원격 저장소)

원격 서버에서 관리하는 저장소를 뜻하며, 협업을 위해 사용됩니다.

## Local Repository (로컬 저장소)

로커 서버에 저장된 저장소를 말합니다.

## Head

현재 커밋한 소스 코드를 뜻합니다.

## Branch

브랜치를 이용하면 코드를 여러개로 복사한 후, 이전의 코드와는 독립적으로 개발할 수 있습니다.

## Push

로컬 저장소의 수정사항을 원격 저장소에 업로드 하는 행위를 뜻합니다.

## Pull

원격 저장소에서 로컬 저장소로 다운로드 하는 행위를 뜻합니다.

## Working Tree

파일 및 폴더를 뜻합니다.

## Index

커밋을 실행하기 전의 저장소와 작업 트리에 존재하는 공간으로, 모든 변경사항이 포함된 파일들의 집합입니다. 따라서, 인덱스에 존재하는 파일 중 선택적으로 커밋할 수 있게 됩니다.
